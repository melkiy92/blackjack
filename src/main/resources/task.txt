Задание:
1) Импортировать проект. Проект должен успешно скомпилироваться. Ознакомиться с описанием игры на вики: http://ru.wikipedia.org/wiki/%D0%91%D0%BB%D1%8D%D0%BA%D0%B4%D0%B6%D0%B5%D0%BA . Мы будем делать самый простой вариант игры - без дополнительных условий и комбинаций.
2) В пакете com.luxoft.dnepr.courses.unit3.model найти класс Card. Вам нужно завершить его имплементацию, добавив поля, указанные в конструкторе и геттеры для них.
3) В том же пакете найдите три энума (перечислимых типа): Rank (значение карты, например - король), Suit (масть) и WinState (состояние игры). Создайте конструктор для Rank, который будет принимать короткое имя карты (2-10,J, Q, K, A) и ее вес (равен значению для цифер, 10 для картинок и 11 для туза). Оба эти значения должны быть доступны через геттеры. Также вы теперь можете с легкостью реализовать метод com.luxoft.dnepr.courses.unit3.model.Card.getCost() без использования ifов.

4) Класс com.luxoft.dnepr.courses.unit3.controller.Deck превратить в утилитный и реализовать оба метода, которые в нем находятся.
Метод createDeck(int size) должен создавать общую колоду карт из нескольких штук. Например для createDeck(1) должна вернуться обычная колода в 52 карты. createDeck(2) возвращает две колоды по 52 карты одна за другой (всего 104 карты). Параметр не может принимать значения меньше 1 и больше 10. Используйте ArrayList для хранения результата. Для того, чтобы лучше изучить как должен работать этот метод (в том числе реакцию на граничные значения) посмотрите на тест com.luxoft.dnepr.courses.unit3.controller.DeckTest.testCreate().
Метод costOf должен возвращать суммарную стоимость списка карт. Реализовать его будет легко, если у вас уже реализован метод Card.getCost().

5) Переходим к реализации com.luxoft.dnepr.courses.unit3.controller.GameController. Обратите внимание что мы можем получить только один экземпляр этого класса. Попробуйте разобраться почему. Документация по каждому методы находится прямо в джавадоке.

6) Реализуем наш интерфейс коммандной строки. Мы уже сделали для вас часть реализации. Ознакомьтесь с ней (пакет com.luxoft.dnepr.courses.unit3.view). Заметьте что классы логики игры и модели ничего не знают о нашем интерфейсе пользователя. Ваша задача в этом задании - реализовать методы в классе com.luxoft.dnepr.courses.unit3.view.CommandLineInterface. Описание методов можно найти в джавадоке. Для проверки вашего интерфейса будет использоваться com.luxoft.dnepr.courses.unit3.view.CommandLineInterfaceTest. Он достаточно сложный - не обязательно полностью его понимать. И все же загляните туда и почитайте джавадоки для него и его суперкласса. Также вы можете теперь запускать игру с помощью com.luxoft.dnepr.courses.unit3.BlackJack как обычное косолькое джава приложение.
